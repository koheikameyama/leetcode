# LeetCode リポジトリ方針

## ディレクトリ構成

```
{番号}-{slug}/
  initial.py     # 初回の解答
  solution.py    # 改善後のベスト解答
  README.md      # 解説
```

## 言語

- 基本は Python
- 他の言語で解いた場合はファイル名で区別（例: `solution.ts`, `initial.ts`）

## README.md の構成

各問題の README.md は以下の順序で書く：

1. 問題タイトル・メタ情報（Difficulty, Tags, URL）
2. 問題概要
3. 初回の解答の問題点
4. 改善した解答（コード + 説明）
5. プロの解答（コード + なぜベストか + 面接での補足）
6. 計算量（時間・空間）
7. 学んだこと

## 解説の言語

- 日本語で書く

## ルートREADME

- 問題を追加するたびに README.md の一覧テーブルを更新すること

## SNIPPETS.md

知らなかった関数・構文に出会ったら [SNIPPETS.md](SNIPPETS.md) に追記する。

- カテゴリ別（`## itertools`, `## collections`, `## string` など）に見出しで整理する
- 各エントリは以下のフォーマットで書く：
  - `### <関数名>`
  - 1行説明
  - 最小サンプルコード
  - **シグネチャ**: `func(arg1, arg2, ...)` の形で記載
  - **引数**: すべての引数の意味・デフォルト値・使いどころを箇条書きで
  - **ポイント**: 使いどころ・類似関数との違いなど
  - Docs: 公式ドキュメントURL
  - 出会った問題: 該当問題ディレクトリへのリンク
